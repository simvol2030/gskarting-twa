## Incident Severity Levels

### P0 - Critical (15-minute response)

**Definition**: Complete service outage, data loss, security breach

**Examples**:
- Backend API down (all stores affected)
- Database corruption
- Security vulnerability exploited
- Cashier app crashes on launch (all stores)

**Contact**: Emergency Hotline + Technical Support (Tier 2/3)

---

### P1 - High (30-minute response)

**Definition**: Partial service degradation, significant impact on operations

**Examples**:
- Transaction success rate <90%
- One store completely non-functional
- Network connectivity issues affecting multiple stores
- 1C integration down (fallback to manual input)

**Contact**: Technical Support (Tier 1) → Backend Team (Tier 2)

---

### P2 - Medium (2-hour response)

**Definition**: Minor issues, workaround available, limited impact

**Examples**:
- QR scanner not working (manual card entry works)
- Slow performance (transactions complete but delayed)
- 1C auto-fetch intermittent (manual input works)
- Single cashier workstation issues

**Contact**: Technical Support (Tier 1)

---

### P3 - Low (Next business day)

**Definition**: Cosmetic issues, feature requests, minor bugs

**Examples**:
- UI display issues (text alignment, colors)
- Non-critical error messages
- Documentation updates
- Training material requests

**Contact**: Technical Support (Tier 1) → Development Team (Tier 3) if bug

## How to Report an Issue

### Step 1: Gather Information

Before contacting support, collect:

1. **Store Information**:
   - Store name and ID
   - Workstation number (if multiple)
   - Date and time of issue

2. **Error Details**:
   - Exact error message (take photo or write down)
   - What you were doing when error occurred
   - Customer ID (if applicable)
   - Transaction amount (if applicable)

3. **Reproduction Steps**:
   - Can you reproduce the issue?
   - Does it happen every time or intermittently?
   - Does it affect all transactions or specific ones?

4. **Impact**:
   - How many transactions affected?
   - Are customers waiting?
   - Can you continue working with a workaround?

---

### Step 2: Contact Appropriate Support Level

**Use this decision tree**:

```
Is the entire system down?
├─ Yes → Emergency Hotline (24/7)
└─ No ↓

Can you complete transactions with a workaround?
├─ Yes → Technical Support (Tier 1) - P2/P3
└─ No ↓

Does it affect only your store?
├─ Yes → Technical Support (Tier 1) - P1
└─ No → Emergency Hotline or Backend Team - P0
```

---

### Step 3: Provide Details

When calling/emailing, state clearly:

1. **Store**: "Store 3 (Сапфир), St. Petersburg"
2. **Severity**: "This is a P1 issue - we cannot process transactions"
3. **Issue**: "QR code scanner not responding, tried restart"
4. **Impact**: "5 customers waiting, cannot earn or redeem points"
5. **Workaround**: "Manual card entry also not working"
6. **Contact**: "Callback number: [phone], Email: [email]"

---

### Step 4: Follow Instructions

Support will:
1. Acknowledge your issue (provide ticket number)
2. Ask clarifying questions
3. Provide troubleshooting steps
4. Escalate if needed
5. Keep you updated on resolution

**Your responsibilities**:
- Follow instructions carefully
- Provide requested information promptly
- Test proposed solutions
- Confirm when issue is resolved

---

### Step 5: Document Resolution

After issue is resolved:
1. Note down ticket number: INC-2025-XXXXX
2. Document workaround (if any) for future reference
3. Update store log with issue and resolution
4. Provide feedback to support if requested
